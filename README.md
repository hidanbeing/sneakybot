# 회사에서 몰래봇 🤖

> 카카오톡 그룹채팅에서 누구나 즐길 수 있는 참여형 미니게임 챗봇

---

## 📌 프로젝트 개요

‘회사에서 몰래봇’은 카카오톡 그룹채팅에서 반복적으로 발생하는  
일상적인 상황(점심 메뉴 선택, 커피 내기, 가벼운 게임 등)을  
**더 재미있고 자연스럽게 해결하기 위해 기획된 실시간 챗봇 서비스**입니다.

평소 친구들과의 그룹채팅에서 챗봇을 활용해 미니게임이나 간단한 의사결정을 경험하며,  
단순한 자동 응답을 넘어 **여러 사용자가 동시에 참여할 수 있는 인터랙티브한 챗봇**의 필요성을 느끼게 되었고,  
이를 바탕으로 본 프로젝트를 개인 프로젝트로 개발하게 되었습니다.

특정 채팅방이나 관리자 권한에 제한되지 않고  
**일반 이용자 누구나 사용할 수 있는 구조**를 목표로 설계했으며,  
엔터테인먼트 요소와 실용성을 동시에 갖춘 서비스 구현에 중점을 두었습니다.

---

## 🛠 기술 스택

- **Runtime**: Node.js
- **Server Framework**: Express
- **Logging**: Morgan
- **Data Management**: JSON 기반 게임 데이터 관리
- **CI / Automation**: GitHub Actions
- **Deployment**: Render

---

## ✨ 주요 기능

### 🎮 미니게임
- **타자 배틀**  
  → 여러 사용자가 동시에 참여하여 타자 입력 속도를 경쟁하는 게임
- **반응속도 테스트**  
  → 랜덤 타이밍 기반의 반응 속도 측정
- **색 몇 개 맞히기**  
  → 랜덤 컬러 인식 기반 미니게임
- **그림 퀴즈**  
  → 제시된 이미지를 보고 정답을 맞히는 참여형 퀴즈
- **헛소리 생성**  
  → 랜덤 텍스트 생성으로 가벼운 재미 제공
- **운세**  
  → 일상적으로 활용 가능한 오늘의 운세 기능

### 🏆 랭킹 시스템
- 타자 배틀 랭킹
- 색 몇 개 랭킹
- 그림 퀴즈 랭킹

게임 결과를 누적 관리하여  
**반복 참여와 경쟁 요소**를 자연스럽게 유도했습니다.

## 📂 프로젝트 구조

```text
.
├─ .github/workflows     # GitHub Actions 설정
├─ color                 # 컬러 게임 로직
├─ debug                 # 디버깅 관련 코드
├─ fallback              # 예외 처리 및 기본 응답
├─ fortune               # 운세 기능
├─ gameData              # 게임 데이터 관리
├─ nonsense              # 헛소리 생성 기능
├─ picture               # 그림 퀴즈
├─ ranking               # 랭킹 관리 로직
├─ typing                # 타자 배틀
├─ utils                 # 공통 유틸 함수
└─ server.js             # 서버 엔트리 포인트
```
기능 단위로 디렉터리를 분리하여
신규 게임 추가 및 유지보수가 용이한 구조로 설계했습니다.

### 🔄 CI / 운영 자동화
Render 환경 특성상 일정 시간 요청이 없을 경우 서버가 비활성 상태로 전환되는 이슈를 확인했습니다.
이를 고려하여 GitHub Actions 기반의 주기적인 헬스 체크 워크플로우를 구성했습니다.

10분 간격으로 /api/ping 엔드포인트에 요청을 보내
서비스 초기 응답 지연을 최소화하고 안정적인 사용자 경험을 유지하도록 설계했습니다.
단순 배포에 그치지 않고, 운영 환경에서 발생할 수 있는 문제를 사전에 고려한 자동화 설정입니다.
